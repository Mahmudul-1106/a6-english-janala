var:
Scope: var is function-scoped, meaning it is available throughout the function where it is declared (or globally if declared outside of a function). It is not block-scoped.
Reassignability: Variables declared with var can be reassigned.
Hoisting: var declarations are hoisted to the top of their scope. This means that the declaration is moved to the top, but the value assignment remains where it is.
let:
Scope: let is block-scoped, meaning it is only available within the block (such as a loop or if statement) where it is declared.
Reassignability: Variables declared with let can be reassigned.
Hoisting: Like var, let is hoisted, but not initialized. This means it enters a "temporal dead zone" where you can't use the variable before it's declared (you’ll get a ReferenceError).
const:
Scope: Like let, const is block-scoped.
Reassignability: Variables declared with const cannot be reassigned. Once a value is assigned to a const variable, it’s fixed.
Hoisting: Like let, const is hoisted but also enters the "temporal dead zone". It must be initialized at the time of declaration.
